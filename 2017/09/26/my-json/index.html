JSON (Javascript Object Notation) js对象表示法
JSON是一种被许多不同系统用于交换数据的一种数据交换格式，类似还有XML数据交换格式。

字面量与变量

字面量(literal)：所谓字面量，是对数据值的具体表示，他的字面意思与其想要表达的意思完全一致，例如

这里中虽然我们没看看到10，但是我们知道x的值已经是10了，其中x就是变量，5就是字面量

JSON是基于JavaScript对象字面量的。注意是“基于”,数据交换格式的核心是数据，所以数据对象是由名称-值对(或者键值-对、属性-值对)来实现的(注意里面不包含函数，因为js对象字面量中常常包含函数，还有一个区别就是js对象里面的名称不需要加引号。这两个区别要牢记)。所以数“基于”。

名称-值对

所谓名称-值对，名称必须加双引号，但值的话，如果是数字、布尔值、数组、对象、null等其他数据类型，这些则不应该加双引号。

一个在线检查JSON格式是否错误的网页：http://www.jsoneditoronline.org/

媒体类型

当你在传输数据时，需要提前告知接收方数据时什么类型，这就涉及到媒体类型(比如互联网媒体类型，内同类型，MIME类型)，他是用”类型/子类型”这种格式来表示，比如见过的text/html。
JSON的MIME类型是application/json

数据类型

所有语言都有三种原始数据类型(或者内置类型)

  • |—— 数字
    　　　——— 整型
    　　　——— 浮点数
    　　　——— 定点数
  • |—— 字符和字符串
  • |—— 布尔类型

大部分语言还有其他数据类型(复合数据类型)，他们是由原始数据类型融合而来的。比如枚举数据类型(数组)，对象数据类型等。

JSON中的数据类型包括

  • 数字

  • 字符串

  • 布尔值

  • 数组

  • 对象

  • null

  • 注意不要把null和js中的undefined混淆，尤其是在使用js时。undefined不是JSON中的数据类型，而在js中，undefined是在尝试获取一些不存在的对象或者变量时返回的结果，也就是说，undefined与那些声明的名称和值都不存在的对象或变量有关,而null则仅仅与对象或变量的值有关。null是一个表示“没有值”的值，在JSON中，null必须使用小写形式*

数组与对象

数组与对象有两个很大的区别：

  1. 数组是值构成的列表或集合，但对象是名称-值对构成的列表或集合
  2. 数组中所有的值应该具有相同的数据类型

XMLHttpRequest

XMLHttpRequest是一种js对象，无需刷新页面即可从一个URL获取数据，常用语AJAX编程

下列是一个get请求

XMLHttpRequest中属性：

  • onreadystatechange
    可以在代码中给他赋值为一个函数

  • readyState
    返回一个0-4的值，用来表示状态吗

  • status
    返回HTTP状态码,(如200表示请求成功)

  • responseText
    当请求成功时，该属性会包含作为文本的相应体(如我们请求的JSON)

注意: js中属性的值可以是一个函数，因为js中函数也是一类对象，对象是一类数据，因此可以被赋值给一个变量(属性)、修改和传递。

request.onreadystatechange表示readyState(就绪状态)每次改变时就会调用该函数。
就绪状态从0-4分别意味着：

  • 0表示未发送: 表示open()函数还没有执行
  • 1表示已经发送: open()函数已经执行，但是send()函数还没有执行
  • 2表示接收到头部: 表示send()函数已执行，切头部和状态码都可以获取了
  • 3表示解析中： 表示头部已经收到，但响应体正在解析中
  • 4表示完成： 表示请求完成，包括响应头和响应体的内容都已经接收到了

跨域资源共享(CORS)

通过设置响应头，是的跨域请求资源(如JSON文档)可以成功。

首先要明白什么是同源策略?
所谓同源策略，就是浏览器处于安全考虑，对资源共享有一定的限制
同源策略要求后台请求仅仅可以请求来自同一域名的资源，由于后台请求仅可以在浏览器的开发者工具中看见，因此保护了普通用户的安全，客户端的则能在网页审查元素中看到。

假如前面实例中的代码是在一个名为http://www.mycoolsite.com的网站上执行的，由于它是最开始的请求的URL，所以它就是原始域名，而API的URL是http://api.openweather.org/data/2.5/weather?lat=35&lon=113,其域名http://api.openweather.org与原始域名http://www.mycoolsite.com不匹配，很明显，这违背了同源策略。所以该API需要使用一些手段来绕过这些现实，才能使后台请求与相应顺利进行。

如何跨域资源共享？
一些API开发者在他们的服务器上实现了跨域资源共享，这些服务器会在响应头额外加上一些带有Access-Control-Allow前缀的属性

例如xxAPI对于请求的http://api.openweather.org/weather?lat=35&lon=113返回的响应头中的Access-Control-Allow部分

这些头部定义了证书是否可用？哪些HTTP(GET、POST、DELETE、HEAD等)是可用的？以及最最重要的一点，即允许哪些域名？本例中，此处包含一个星号*，他表示任何域名都是允许的。

CORS使得如Open Weather这样的站点可以让用户在用户端使用AJAX交互的方式获取他们的数据，同事，CORS还可以用于禁止某些域名的访问，来防止有些人用API做坏事，如CSRF，eg：黑客企图通过将JSON放入标签来利用银行站点的信任。现在银行有了一个新的安全方式来防止这种攻击，这个方式就是实现CORS并且在其响应头中加入类似如下的内容：

JSON-P

使用script标签，绕过同归策略的限制，以从不同域名的服务器上请求JSON

JSON-P指带有padding的JSON,我们之前说的标签不受同源策略的影响，JSON-P就是利用这一点来向不同域名的站点请求JSON的
JSON-P并没有CORS那么理想，它只是一个备选方案(CORS是更好的方案，同时也是一种标准)，不过有时候，还是很需要这种不是很规范的解决方案的。当不能使用CORS时，仍需要通过某种关系来实现http://www.domainone.com和http://www.domaintwo.com之间的交流。
